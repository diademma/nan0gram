# Nan0gram Messenger UI & Android WebView Bridge — Технический путеводитель

Данный проект содержит интерфейс мессенджера React и системный мост взаимодействия с ОС Android (Kotlin) через WebView.

---

## 1. Структура директорий проекта

Модули распределены по двум изолированным папкам в корне активов WebView:

    📁 app/src/main/assets/
    │
    ├── 📁 n0g_messenger_ui/              # [Папка 1] Фронтенд-интерфейс чата на React (ESM)
    │   ├── 📄 n0g_msg_utils_101.js
    │   ├── 📄 n0g_msg_bridge_102.js
    │   ├── 📄 n0g_msg_settings_view_103.js
    │   ├── 📄 n0g_msg_ui_modals_104.js
    │   ├── 📄 n0g_msg_chat_area_105.js
    │   └── 📄 n0g_msg_app_entry_106.js
    │
    ├── 📁 n0g_bridge/                    # [Папка 2] Системный мост связи с Kotlin (Global Scripts)
    │   ├── 📄 bridge-recorder.js
    │   ├── 📄 bridge-core.js
    │   └── 📄 bridge-player.js
    │
    ├── 📄 cipher.js                      # Библиотека шифрования (Global Script)
    ├── 📄 react-lib.js                   # Ядро библиотеки React (Global Script)
    └── 📄 index.html                     # Точка входа для WebView

---

## 2. Схема загрузки скриптов и подчинения

Скрипты в `index.html` должны загружаться в строгой последовательности для сохранения цепочки глобальных зависимостей:

1. **`react-lib.js`** — предоставляет глобальное ядро React (`window.React`).
2. **`cipher.js`** — предоставляет методы шифрования (`window.nanoCipher`).
3. **`n0g_bridge/bridge-recorder.js`** — независимый глобальный обработчик жестов записи звука.
4. **`n0g_bridge/bridge-core.js`** — ядро системного моста (требует `cipher.js`, регистрирует `window.nan0gram`).
5. **`n0g_bridge/bridge-player.js`** — MutationObserver для аудиоплеера (активируется после загрузки DOM).
6. **`n0g_messenger_ui/n0g_msg_app_entry_106.js`** — точка входа React-интерфейса (ES-модуль, импортирует модули `101`–`105`, связывается с мостом `102` и вешает реакцию на события из `bridge-core`).

## 3. Описание модулей нативного моста (Папка `n0g_bridge/`)

### 📄 bridge-recorder.js
* **Тип:** Чистый JS (ES6). Глобальный контекст.
* **Ответственность:** Интерфейс и оверлей записи голоса. Обработка длинного удержания (long-press) кнопки микрофона, анимации REC-бара с таймером, кнопки отмены записи, защита от случайных кликов и перехват аудиофокуса нативного микрофона Android.
* **Внутренние зависимости:** Нет. Полностью автономен.

### 📄 bridge-core.js
* **Тип:** Чистый JS (ES6). Глобальный контекст.
* **Ответственность:** Ядро моста и криптография. Настройка фонового Stealth Composer (поиск активного инпута на странице, генерация сессионных AES/RSA ключей, шифрование текста и авто-отправка). Предоставляет глобальный объект `window.nan0gram`. Слушает события от Kotlin: `email-received` (с дешифрацией AES/RSA конвертов и авто-кэшем), `local-media-sent` и др.
* **Место расширения:** Содержит изолированные методы `_buildBody()` и `submitMedia()`, куда встроится будущая стратегия быстрого отключения шифрования и простой обфускации метаданных.
* **Внутренние зависимости:** Требует предварительной загрузки `cipher.js` в глобальную область видимости.

### 📄 bridge-player.js
* **Тип:** Чистый JS (ES6). Глобальный контекст.
* **Ответственность:** Кастомный Telegram-подобный плеер голосовых сообщений. `MutationObserver` для автоматического развертывания плеера поверх тегов `.voice-msg` и поддержка перемотки жестами по волновой диаграмме.
* **Внутренние зависимости:** Нет. Инициализируется на событии `DOMContentLoaded`.

## 4. Описание модулей мессенджера (Папка `n0g_messenger_ui/`)

### 📄 n0g_msg_utils_101.js
* **Тип:** Чистый JS (ES6). Не содержит React-кода, JSX или обращений к DOM.
* **Ответственность:** Статические данные (заглушки контактов, темы, эмодзи) и чистые функции-хелперы (расчет сетки альбомов картинок, форматирование длительности аудио, расчет безопасных координат контекстного меню).

### 📄 n0g_msg_bridge_102.js
* **Тип:** Чистый JS (ES6). Не содержит React-кода.
* **Ответственность:** Безопасный интерфейс-фасад (адаптер) для обращения React-слоя к методам `bridge-core.js` и Kotlin. Предотвращает ошибки («вылеты») UI при отладке в обычном браузере на ПК.

### 📄 n0g_msg_settings_view_103.js
* **Тип:** React UI-модуль.
* **Ответственность:** Компоненты настроек (`SettingsPanel` — оформление, приватность никнейма, очистка нативного кэша) и тем оформления (`ThemePanel` — прозрачность и цвет баблов чата).

### 📄 n0g_msg_ui_modals_104.js
* **Тип:** React UI-модуль.
* **Ответственность:** Компоненты левой панели со списком чатов (`Sidebar`), карточки профиля (`ProfileModal`), полноэкранной галереи просмотра медиа (`Lightbox` с pinch-to-zoom жестами) и тостов (`Toast`).

### 📄 n0g_msg_chat_area_105.js
* **Тип:** React UI-модуль.
* **Ответственность:** Область переписки. Содержит рендеринг строки сообщения (`MessageRow`), контекстное всплывающее меню действий (`ContextMenu`) и центральную панель диалога (`ChatArea` — поле ввода, нативная запись звука, закрепленные сообщения).

### 📄 n0g_msg_app_entry_106.js
* **Тип:** Точка входа React (React-DOM монтирование).
* **Ответственность:** Инициализация мессенджера, хранение корневого React-стейта, монтирование дерева компонентов в DOM и реакция на системные события Котлина.
* **Экспортирует:** По умолчанию компонент `AppRoot`. Монтирует дерево в тег `<div id="root">`.

## 5. Совместимость с React-бандлом

Для работы скомпилированных React-вызовов (`T.useState`, `f.jsx`) напрямую в WebView без сборщиков, файлы настроек (103), модалок (104), чата (105) и точки входа (106) содержат в самом начале совместимый заголовок для маппинга глобальных переменных:

    const T = window.T || window.React;
    const Q0 = window.Q0 || window.ReactDOM || window.ReactDom; // только в app_entry_106
    const f = window.f || {
        jsx: (type, props, key) => T.createElement(type, { ...props, key }),
        jsxs: (type, props, key) => T.createElement(type, { ...props, key })
    };

Этот заголовок перенаправляет вызовы на глобальную оптимизированную библиотеку `react-lib.js`. При доработке UI-компонентов модифицировать или удалять этот заголовок запрещено.
