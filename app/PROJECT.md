# Nan0gram Messenger UI — Путеводитель по проекту

Данный модуль отвечает за интерфейс мессенджера React, работающий в WebView Android.

* **Директория файлов:** `app/src/main/assets/n0g_messenger_ui/`
* **Префикс названий:** `n0g_msg_`
* **Способ подключения:** ES-модули (ESM). Точка входа подключается в `index.html` через `<script type="module" src="n0g_messenger_ui/n0g_msg_app_entry_106.js"></script>`.

---

## 1. Схема подчинения и импортов

Зависимости между модулями строго однонаправленные (направленный ациклический граф). Модули верхнего уровня импортируют данные из нижних уровней.

* `n0g_msg_utils_101.js` (Базовый независимый модуль)
* `n0g_msg_bridge_102.js` (Базовый независимый модуль)
* `n0g_msg_settings_view_103.js` (Зависит от: `101`, `102`)
* `n0g_msg_ui_modals_104.js` (Зависит от: `102`, `103`)
* `n0g_msg_chat_area_105.js` (Зависит от: `101`, `102`)
* `n0g_msg_app_entry_106.js` (Зависит от: `101`, `102`, `104`, `105` — корневой файл)

---

## 2. Описание модулей и зоны ответственности

### 📄 n0g_msg_utils_101.js
* **Тип:** Чистый JS (ES6). Не содержит React-кода, JSX или обращений к DOM.
* **Ответственность:** Статические константы (эмодзи, темы по умолчанию, заглушки чатов) и чистые функции-хелперы.
* **Экспортирует:** `DEFAULT_AVATAR_URL`, `INITIAL_CHATS_FALLBACK`, `QUICK_REACTIONS`, `ALL_REACTIONS`, `DEFAULT_THEME`, `hslToRgb()`, `getAlbumLayout()`, `formatDuration()`, `calculateMenuPosition()`, `getCurrentTime()`, `getMessagePreviewText()`, `generateUniqueId()`.

### 📄 n0g_msg_bridge_102.js
* **Тип:** Чистый JS (ES6). Не содержит React-кода.
* **Ответственность:** Изолированный отказоустойчивый слой взаимодействия с нативными методами Android Kotlin. Предотвращает ошибки (краши) при отсутствии моста.
* **Экспортирует:** `getDeviceId()`, `getSettingString()`, `clearMediaCache()`, `clearAllHistoryLog()`, `saveChatToDb()`, `requestChatsList()`, `requestChatHistory()`, `saveMessageToDb()`, `deleteMessageFromDb()`, `updateMessageReactionInDb()`, `setWallpaperPending()`, `saveMediaToDownloads()`, `submitStealthFile()`, `submitBase64Media()`.

### 📄 n0g_msg_settings_view_103.js
* **Тип:** React UI-модуль.
* **Ответственность:** Интерфейс панелей кастомизации темы оформления и настроек.
* **Экспортирует:** 
  * `ThemePanel` — выбор прозрачности и цвета (оттенка) чата.
  * `SettingsPanel` — экран настроек (Оформление, Приватность с изменением никнейма, Управление нативным кэшем).

### 📄 n0g_msg_ui_modals_104.js
* **Тип:** React UI-модуль.
* **Ответственность:** Боковая панель и вспомогательные глобальные модальные компоненты.
* **Экспортирует:** 
  * `Sidebar` — список чатов, баджи непрочитанных, реакция на свайпы, вывод оверлеев настроек и темы чата.
  * `ProfileModal` — карточка просмотра информации о контакте.
  * `Lightbox` — полноэкранная галерея (pinch-to-zoom жесты, выбор скачивания фото или всего альбома).
  * `Toast` — всплывающие системные уведомления.

### 📄 n0g_msg_chat_area_105.js
* **Тип:** React UI-модуль.
* **Ответственность:** Главная область переписки, сообщения и контекстное меню.
* **Экспортирует:** 
  * `MessageRow` — строка сообщения (текст, фотоальбомы, видео/голосовые плееры, файлы, реакции, свайп-ответ и лонгпресс).
  * `ContextMenu` — всплывающее меню действий и реакций.
  * `ChatArea` — диалоговое окно, поле ввода, запись аудио через MediaRecorder, закрепленные сообщения.

### 📄 n0g_msg_app_entry_106.js
* **Тип:** Точка входа приложения.
* **Ответственность:** Инициализация мессенджера, хранение корневого React-стейта, монтирование дерева компонентов в DOM и реакция на системные события Котлина.
* **Экспортирует:** По умолчанию компонент `AppRoot`. Монтирует дерево в тег `<div id="root">`.

## 3. Совместимость с React-бандлом
Для работы скомпилированных React-вызовов (`T.useState`, `f.jsx`) напрямую в WebView без сборщиков, файлы настроек (103), модалок (104), чата (105) и точки входа (106) содержат в самом начале совместимый заголовок для маппинга глобальных переменных:

    const T = window.T || window.React;
    const Q0 = window.Q0 || window.ReactDOM || window.ReactDom; // только в app_entry_106
    const f = window.f || {
        jsx: (type, props, key) => T.createElement(type, { ...props, key }),
        jsxs: (type, props, key) => T.createElement(type, { ...props, key })
    };

Этот заголовок перенаправляет вызовы на глобальную оптимизированную библиотеку `react-lib.js`. При доработке UI-компонентов модифицировать или удалять этот заголовок запрещено.
